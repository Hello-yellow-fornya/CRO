# QA Checklist

## Pre-Launch QA

Use this checklist before launching any A/B test or site change.

### 1. Hypothesis & Setup
- [ ] Hypothesis documented with expected outcome and primary KPI
- [ ] Test tagged correctly in the testing platform (name, campaign, audience)
- [ ] Traffic allocation set correctly (e.g., 50/50)
- [ ] Targeting rules verified (URL, audience, device)

### 2. Visual & Functional
- [ ] Variation renders correctly on desktop
- [ ] Variation renders correctly on tablet
- [ ] Variation renders correctly on mobile
- [ ] No layout shifts or broken elements
- [ ] All CTAs / links work correctly
- [ ] Forms submit correctly (if applicable)
- [ ] Dynamic content / personalisation displays correctly

### 3. Tracking & Analytics
- [ ] Primary goal tracking fires on the correct event
- [ ] Revenue / transaction tracking is accurate (if applicable)
- [ ] No duplicate event firing
- [ ] GA / analytics integration verified
- [ ] Custom dimensions or events tagged (if applicable)

### 4. Cross-Browser
- [ ] Chrome
- [ ] Safari
- [ ] Firefox
- [ ] Edge
- [ ] iOS Safari
- [ ] Android Chrome

### 5. Performance
- [ ] No significant page speed regression
- [ ] No console errors introduced
- [ ] Flicker prevention in place

### 6. Sign-Off
- [ ] Internal team review complete
- [ ] Client approval received (if required)
- [ ] Launch date and monitoring plan confirmed
