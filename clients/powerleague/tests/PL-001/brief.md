# Test Brief — PL-001

**Client:** Powerleague
**Page:** /football-birthday-party
**Test size:** BIG
**Date:** 2026-03-19

## Hypothesis

If we restructure the Kids Party landing page to surface location first (before packages), move the 93% recommendation stat into the hero, and reduce package cognitive load by collapsing feature lists, then booking flow starts (begin_checkout) will increase because the two primary conversion barriers — unresolved location anxiety and decision paralysis from exhaustive package lists — are removed before the user reaches the commitment point.

**Evidence:**
- Knowledge base: location-first is highest-evidence principle for leisure/activity booking (+15–30% form starts)
- Known friction point #1: location not surfaced until after package selection
- Known friction point #4: 93% recommendation stat buried below fold
- Known friction point #5: package feature lists exhaustive — high cognitive load

**Primary KPI:** begin_checkout rate
**Secondary KPI:** package_selected event (which package)
**MDE target:** 15% relative
**Estimated duration:** 6 weeks at current traffic

## Changes from control

1. Hero: postcode finder replaces dual CTAs as primary action
2. Hero: "93% of parents recommend us" moved to hero subheadline
3. Hero: single dominant CTA removed — postcode entry IS the CTA
4. Packages: feature lists collapsed to 4 bullets max, "See all" expand
5. Packages: emotional benefit leads each package card
6. Split payment: surfaced at package selection, not checkout
7. Urgency: "Next available [month]" shown after postcode entry
